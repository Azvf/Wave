# 架构决策记录 (Architecture Decision Records)

## 什么是 ADR？

架构决策记录（ADR）是一种记录架构决策的文档格式，用于记录：
- **为什么选择某个方案**（而非仅仅记录选择了什么）
- **考虑了哪些替代方案**
- **做出决策的背景和理由**
- **决策带来的后果和影响**

## ADR 格式

每个 ADR 文件应包含以下部分：

1. **状态** (Status): 已采用 / 已弃用 / 已替代
2. **背景** (Context): 做出决策的背景和问题描述
3. **考虑的选项** (Options): 评估的所有可行方案
4. **决策** (Decision): 最终选择的方案
5. **理由** (Rationale): 为什么选择这个方案，关键考虑因素
6. **后果** (Consequences): 决策带来的影响、风险和注意事项

## 如何创建新的 ADR

1. 使用命名格式: `ADR-XXX-简短描述.md`
2. 按照 ADR 格式填写所有必要部分
3. 更新本 README 的索引列表
4. 更新 [文档索引](../文档索引.md) 中的 ADR 章节
