# 鼠标交互

**核心原则：Hover 反馈 (Hover Feedback) · 光标感知 (Cursor Awareness) · 上下文操作 (Contextual Actions)**

## Hover 态

**规则**: 桌面端必须有明确的 Hover 反馈。

**实现**:
- 使用 `MouseRegion` 监听鼠标悬停
- 提供视觉反馈（颜色变化、阴影、缩放等）

## 光标感知

**规则**: 按钮上使用 `SystemMouseCursors.click`，拖拽区使用 `resize`。

**光标类型**:
- **按钮**: `SystemMouseCursors.click`
- **可拖拽**: `SystemMouseCursors.grab` / `SystemMouseCursors.grabbing`
- **可调整大小**: `SystemMouseCursors.resizeLeftRight` / `SystemMouseCursors.resizeUpDown`
- **文本输入**: `SystemMouseCursors.text`
- **禁用**: `SystemMouseCursors.forbidden`

**实现**: 使用 `MouseRegion` 的 `cursor` 属性

## 上下文操作

**规则**: 针对桌面端优化交互密度。

- **悬停显示**: 列表项悬停时显示辅助操作（如删除按钮、编辑图标）
- **右键菜单**: 支持右键上下文菜单（使用 `ContextMenuButton`）

## 响应式布局

**规则**: 桌面端窗口大小多变，必须处理断点。

**断点定义**:
- **Compact**: < 600px (隐藏次要图标，缩小 Padding)
- **Standard**: 600px - 1200px (标准布局)
- **Wide**: > 1200px (利用横向空间，如两列布局)

**约束管理**: 为所有卡片设置 `BoxConstraints(maxWidth: 800)`，防止在大宽屏上内容过度拉伸。

详见: [窗口管理](./窗口管理.md) | [键盘优先](./键盘优先.md)
