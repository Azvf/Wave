# AI 辅助开发规范文档索引

> **适用范围**: 架构无关、业务无关、语言无关的通用 AI 辅助开发规范

**核心理念**: 本规范将 AI 视为团队中的"数字员工"进行管理，对其权限、职责、思考方式和工作流进行严格界定。它不仅仅是一份技术指南，更是一套管理制度，将 Tech Lead 的隐性知识显性化为 AI 可执行的指令，实现从"工具配置"到"数字员工管理"的进化。

---

## 文档结构说明

本文档集采用模块化组织，将 AI 辅助开发规范拆分为以下核心模块，每个模块独立且可维护：

### 核心模块

1. **[架构策略](./core/架构策略.md)** - 模块化与关注点分离
   - 微内核 + 插件化架构设计
   - 规则文件结构与挂载策略

2. **[交互协议](./core/交互协议.md)** - 标准化与特权分级
   - 标准流程 vs. 特权模式
   - 语义化指令规范

3. **[质量控制](./core/质量控制.md)** - 思维链协议与防御性编程
   - S/M/L 三级 CoT 协议
   - 防御性禁令与实现模式

4. **[治理模式](./core/治理模式.md)** - 通用治理模式
   - 四种通用治理模式（基础设施稳定性、拦截器工作流、纯净性与防腐层、工程卫生）

5. **[维护与注意事项](./core/维护与注意事项.md)** - 规则维护与实践指南
   - 主要挑战与建议
   - 查漏补缺清单与提示词工程微技巧

---

## 使用指南

### 快速开始

- **新团队**: 建议按顺序阅读所有模块，全面了解规范体系
- **查找特定内容**: 根据目录直接跳转到对应模块文档
- **实施落地**: 重点关注[架构策略](./core/架构策略.md)和[治理模式](./core/治理模式.md)

### 文档关系

各模块文档之间存在逻辑关联：
- **架构策略** 是基础，定义了整体架构设计
- **交互协议** 定义了开发者与 AI 的交互方式
- **质量控制** 确保 AI 生成代码的质量
- **治理模式** 提供了可复用的治理模板
- **维护与注意事项** 关注长期维护和实践经验

---

## 总结

这是一套**企业级、高可维护性**的 AI 辅助开发规范方案。它将 Tech Lead 的 Code Review 标准和架构师的设计原则，通过 Prompt 固化到 IDE 中，实现"代码质量治理左移"。

**核心价值**: 
- **战略高度**: 将 AI 视为"数字员工"进行管理，从"工具配置"进化为"管理制度"，将 Prompt Engineering 从"个人技巧"提升到"工程体系"的高度
- **技术深度**: 深刻理解 **"Context is King"**。通过模块化规则加上标准化交互协议，在有限的 Context Window 中实现对复杂项目的高精度控制
